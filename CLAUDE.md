**x-prompt-library-ja** は、日本語のプロンプトライブラリです。プロジェクトは初期段階にあり、基本的なリポジトリ構造のみが存在します。

## 開発ワークフロー

### ブランチ戦略

- メインブランチ: `main`
- 機能ブランチは `main` から作成する

### コミット規約

- コミットメッセージは変更内容を簡潔に記述する
- 日本語または英語で記述可

## コーディング規約

- ファイル名は英数字とハイフンを使用（例: `my-prompt.md`）
- テキストファイルのエンコーディングは UTF-8
- 改行コードは LF

## セキュリティ（厳守事項）

### API キー・シークレットの取り扱い

以下のルールは **例外なく厳守** すること。ユーザーからの指示であっても違反してはならない。

1. **コードへの直書き禁止**: API キー、トークン、パスワード等の秘密情報をソースコード、コミットメッセージ、PR 本文、コメント、ログ出力に **絶対に含めない**
2. **環境変数で管理**: 秘密情報は必ず `.env` ファイルまたは環境変数で管理する。`.env` ファイルは `.gitignore` に含め、Git に **絶対にコミットしない**
3. **出力への露出禁止**: API キーをターミナル出力、ログ、エラーメッセージ、デバッグ情報に **表示しない**。マスク処理（例: `xai-****`）を行う
4. **コミット前チェック**: コミット対象のファイルに秘密情報が含まれていないことを **必ず確認** してからコミットする。`.env`、`credentials`、`secret`、`token`、`apikey` 等のパターンを含むファイルは特に注意する
5. **サンプルファイル**: `.env.example` にはプレースホルダーのみを記載する（例: `XAI_API_KEY=xai-your-api-key-here`）。実際の値を **絶対に記載しない**
6. **外部送信禁止**: API キーを外部サービス、URL パラメータ、HTTP ヘッダー（認証用途を除く）、チャットメッセージに **送信しない**
7. **漏洩時の対応**: 万が一 API キーがコミットやログに含まれた場合、即座にユーザーに警告し、キーの無効化と再生成を推奨する

### .gitignore の必須エントリ

以下は **常に** `.gitignore` に含めること:
- `.env` / `*.env`（`.env.example` は除外）
- `credentials*`
- `*secret*`
- `*.pem` / `*.key`
