## 2. 即金レイヤー①: 技術顧問・スポットコンサル

### 2-A. 技術顧問（月額固定）

最も時間効率が高い収入源。シニアレベルのスキルがあれば最も早く月収の基盤を作れる。

| 項目 | 内容 |
|------|------|
| **業務内容** | 週1回のMTG（1-2時間）+ Slack/チャットでの技術相談対応 |
| **単価相場** | 月15-30万円/社 |
| **時間コスト** | 実質 週2-3時間/社 |
| **契約形態** | 業務委託（月額固定） |

**具体的な提供価値（何を売るか）:**
- アーキテクチャレビュー・設計相談
- コードレビュー（PR単位）
- 技術選定のアドバイス
- AI/バイブコーディング導入支援（今なら需要が爆発中）
- セキュリティ・パフォーマンス改善の指針

**獲得方法:**
1. **既存の人脈** — 過去の同僚・取引先に「技術顧問やってます」と声をかける
2. **X上での発信** — 技術記事・知見を投稿してインバウンド獲得
3. **プラットフォーム経由:**
   - [MENTA](https://menta.work/) — メンタリング・顧問マッチング
   - [ビザスク](https://visasq.co.jp/) — スポットコンサル（1時間1-5万円）
   - [Contact EARTH matching](https://dx-consultant.co.jp/) — DX案件特化、高単価
   - [Findy Freelance](https://freelance.findy-code.io/) — モダン技術・高単価

**目標:** 1-2社の技術顧問契約で **月15-25万円** を安定確保

## 3. 即金レイヤー②: バイブコーディング受託

### なぜ今「バイブコーディング受託」が最強か

1. **需要が爆発中** — 「バイブコーディング」検索が6,700%増。企業も個人も「AIでアプリ作りたい」と思っているが、やり方がわからない
2. **シニアスキル × AI = 圧倒的な生産性** — ジュニアがAIを使うのとシニアがAIを使うのでは品質が全く違う。設計力があるからAIの出力を正しく評価・修正できる
3. **制作スピード = 実質時給の爆上げ** — 従来2-4週間の案件をAI活用で3-5日で納品可能。時給換算で大幅に上がる
4. **ストーリーになる** — 「シニアエンジニアが教えるAI活用開発」は発信コンテンツとしても価値が高い

### 3-A. LP・Webサイト制作（入門編・最速で稼げる）

| 項目 | 内容 |
|------|------|
| **単価** | 5-15万円/件 |
| **制作時間** | 3-8時間（Lovable/Bolt.new + Cursor仕上げ） |
| **実質時給** | 1-3万円 |
| **ターゲット** | 個人事業主、スタートアップ、副業者 |
| **月間件数目標** | 2-4件 |

**受注チャネル:**
- X（Twitter）で制作過程をBuild in Publicで投稿 → DM受注
- ココナラ出品（「AIでLP制作します」）
- CrowdWorks/Lancersでプロジェクト応募

---

### 3-B. 業務ツール・ミニアプリ制作（本命・高単価）

| 項目 | 内容 |
|------|------|
| **単価** | 15-50万円/件 |
| **制作時間** | 20-40時間（1-2週間） |
| **実質時給** | 1-2万円 |
| **ターゲット** | 中小企業、個人事業主 |
| **具体例** | 予約システム、顧客管理、在庫管理、問い合わせBot |

**なぜ中小企業向けが狙い目か（前回分析の再確認）:**
- 日本の中小企業DX実施率はわずか **4.6%**
- 「費用が高い」「やり方がわからない」が主な障壁
- 従来の開発会社に頼むと100-500万円 → バイブコーディングで15-50万円なら圧倒的に安い
- 地方では相談先すら不足

**受注フロー:**
```
X投稿で認知獲得
    ↓
DMまたはフォームで相談受付
    ↓
30分の無料ヒアリング（Zoom）
    ↓
翌日に簡易プロトタイプを提示（Lovable/Boltで30分で作る）← ここがキラーポイント
    ↓
見積もり提示 → 受注
    ↓
1-2週間で納品
    ↓
保守契約（月額1-3万円）に繋げる ← ストック収入化
```

**キラーポイント: プロトタイプ即日提示**

従来の受託開発では、要件定義→見積もり→発注→開発→納品に数ヶ月かかる。
バイブコーディングなら**ヒアリング翌日に動くプロトタイプを見せられる**。
このスピード感が中小企業オーナーにとって驚異的で、成約率が大幅に上がる。

---

### 3-C. AI機能組み込み受託（高単価・差別化）

| 項目 | 内容 |
|------|------|
| **単価** | 30-100万円/件 |

| **ターゲット** | スタートアップ、DX推進中の中堅企業 |

シニアレベルのスキルがあるからこそ取れる案件。設計・セキュリティ・スケーラビリティまで対応できることが差別化ポイント。

### 3-D. 保守契約でストック化

| 項目 | 内容 |
|------|------|

| **業務内容** | 軽微な修正対応、サーバー監視、月1回の定例MTG |
| **時間コスト** | 月1-2時間/社 |
| **目標** | 10社で月10-30万円のストック収入 |

## 4. ストック型レイヤー: テンプレート・デジタルプロダクト販売

受託で稼ぎながら、同時に「寝ている間も売れる」資産を構築する。

### 4-A. 開発テンプレート販売

| 項目 | 内容 |
|------|------|
| **販売物** | Next.js SaaSスターターキット、LP テンプレート、管理画面テンプレート等 |
| **単価** | 3,000-30,000円 |
| **販売先** | Gumroad, Lemon Squeezy, BOOTH, note |
| **制作時間** | 20-40時間（1回作れば何度でも売れる） |

**なぜテンプレートか:**
- 受託案件で作ったものをテンプレート化 → 2度美味しい
- 「バイブコーディングのスターターキット」は今まさに需要が急増中
- 海外のMakerKitは$299で販売して大きな収益を上げている

**狙い目のテンプレート:**
1. **日本語SaaSスターターキット**（認証・決済・管理画面・LINEログイン付き）
2. **中小企業向け予約システムテンプレート**（受託案件の汎用化）
3. **AI機能付きチャットBotテンプレート**（RAG構成済み）
4. **LP高速制作テンプレートセット**（業種別 × 5-10種類）

---

### 4-B. 技術コンテンツ販売

| 項目 | 内容 |
|------|------|
| **販売物** | 「シニアエンジニアのバイブコーディング実践ガイド」等 |
| **単価** | 500-3,000円 |
| **販売先** | note（有料記事）、Zenn（本）、Gumroad |

受託案件で得たノウハウを体系化してコンテンツ化する。

---

### 4-C. Micro-SaaS（中長期）

受託で見つけた「何度も同じ要望が来る」ニーズをプロダクト化。

| 項目 | 内容 |
|------|------|
| **月額** | 980-9,800円/ユーザー |
| **例** | 前回分析の案F-M（AI接客トレーニング、パーソナルカラー診断、シフト管理等） |
| **目標** | 月5-30万円（半年-1年後） |

これはグルメスワイプと並行で育てる長期資産。急がない。

## 5. X運用と連動した案件獲得フロー

全ての収入レイヤーを**X（Twitter）からの集客**で回す。

### 投稿カテゴリと頻度

| カテゴリ | 例 | 頻度 |
|---------|-----|------|
| **Build in Public** | 「今日はクライアントの予約システムを3時間で作った。Cursor × Claude Codeの組み合わせが最強」 | 毎日1-2回 |
| **受託実績** | 「飲食店オーナーに予約管理ツールを納品。FAXから脱却して毎月3時間の作業が消えた」 | 週2-3回 |
| **技術知見** | 「日本語RAGで精度を上げるコツ3選」「Lovableで作ったLPをCursorで本番品質にする方法」 | 週2-3回 |
| **収益公開** | 「副業3ヶ月目の収益: 受託38万 + テンプレ5万 + 顧問20万 = 63万」 | 月1回 |
| **グルメスワイプ進捗** | メインプロダクトの開発進捗もBuild in Publicで共有 | 週1-2回 |

### 投稿→受注の導線

```
X投稿（技術知見・制作実績）
    ↓
プロフィールに「AI活用のWeb制作・ツール開発承ります」
    ↓
固定ポストに「制作実績まとめ + 料金目安 + 相談フォームリンク」
    ↓
DM or フォーム経由で相談
    ↓
無料ヒアリング → プロトタイプ提示 → 受注
```

**ポイント:** 「非エンジニア向けにわかりやすく書く」のがX集客の鉄則。
技術者同士の交流ではなく、**ターゲットは「ツールが欲しい非エンジニア」**。

## 8. グルメスワイプとのシナジー設計

食い繋ぎの仕事とグルメスワイプは**相互に強化し合う関係**にできる。

### シナジー①: 技術スタックの共通化

グルメスワイプで使っている技術スタック（React/Next.js、API設計、DB設計等）をそのまま受託案件でも使う。
→ 受託で得た知見がグルメスワイプの品質向上に直結。

### シナジー②: Build in Publicの一本化

X上の発信を「グルメスワイプの開発」と「受託案件の実績」の両方を混ぜて投稿。
→ 「このアプリを作っている人が受託もやっている」という信頼感。
→ グルメスワイプのファン = 受託の潜在顧客、という循環。

### シナジー③: 飲食業界の人脈

グルメスワイプは飲食関連のアプリ。受託で飲食店向けツール（予約・メニュー管理・注文システム等）を作れば：
- 飲食業界の人脈が両方に活きる
- 受託クライアントがグルメスワイプの初期ユーザーになりうる
- 業界理解が深まり、プロダクトの質が上がる

### シナジー④: 受託で見つけたペインをプロダクト化

受託で「この要望、何度も聞くな」というパターンが見つかったら、それをMicro-SaaSとしてプロダクト化。
→ グルメスワイプ以外の収益資産にもなる。
