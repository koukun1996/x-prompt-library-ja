# プロフィールCTA最適化案（GourmetSwipe / @GourmetSwipe）

最終更新: 2026-02-12
調査ベース: Xアルゴリズム2026（Phoenixモデル）、高フォロー率アカウントのCTA分析

---

## 現状の問題点

```
【現在の名前】
PC坊主｜タイパ重視の次世代グルメアプリ開発中

【現在のBio】
「店選びの破壊」を目指す開発者 | 検索疲れにサヨナラ |
縦型動画を眺めるだけで最高の店が見つかる『グルメスワイプ』作ってます |
タイパ重視の次世代グルメアプリ | 文字情報だけの検索はもう古い |
8年目エンジニアが本気で作る最高のUX体験をあなたに |
開発のリアルと最新情報はここから↓
```

### 致命的な問題5つ

1. **0.5秒ルール不適合**: プロフを訪れた人が「誰？何の人？フォローする価値ある？」を瞬時に判断できない
2. **名前の「PC坊主」がグルメと繋がらない**: 投稿テーマ（グルメ）と名前が乖離 → 記憶に残らない
3. **企業プレスリリース調**: 「最高のUX体験をあなたに」は山田の文体と真逆。来た人が投稿とのギャップで離脱する
4. **CTA（行動喚起）が弱い**: 「開発のリアルと最新情報はここから↓」はフォローする動機にならない
5. **フォロー → LINE の2段階導線がない**: フォローを先にさせ、LINEは固定ポスト or リプライで誘導すべき

---

## 改善の設計思想

### 導線設計（2段階CTA）

```
【ステップ1】ポストを見る → プロフィールに来る
  └→ プロフィールが「0.5秒で」刺さる → フォロー

【ステップ2】フォロー後、固定ポストを見る → LINE登録
  └→ 固定ポストに「LINEでしか得られない情報」を提示 → LINE登録
```

**プロフィール本文にLINEリンクを貼らない。**
理由: プロフィールの役割は「フォローさせること」に集中させる。LINEは固定ポストで担当。

### プロフィール設計の3原則（2026年アルゴリズム対応）

1. **名前 = 検索キーワード + 記憶フック**（15文字以内）
2. **Bio = 「誰に何を」+ 実績/信頼 + フォロー理由**（160字以内）
3. **リンク = LINE公式（1箇所のみ）**

## 改善案 A（推奨）: 共感型 × 個人開発者ブランディング

### 名前
```
山田｜グルメスワイプ開発中🍜
```

**理由:**
- 「山田」= 親しみやすさ + 人感（企業感を消す）
- 「グルメスワイプ」= アプリ名の認知を積む
- 「開発中」= 応援したくなる未完成感
- 🍜 = 視覚フック（タイムラインで目立つ）
- 15文字以内で検索にも引っかかる

### Bio
```
飯探しに30分かけてたエンジニアが、3秒で店が見つかるアプリ作ってる🔥

動画スワイプで「今日ここ行こ」が決まる
▸ 開発のリアル / グルメ / AIコーディング
▸ 3月ベータ版リリース予定

店選びで消耗してる人、フォローしといて
```

**設計の解説:**
- **1行目**: 「30分 → 3秒」の数字対比でフック（ポストと同じ手法）
- **2行目**: アプリの価値を1文で。「TikTok × 食べログ」より体験で伝える
- **3行目**: 発信テーマを明示 → フォロー後に何が得られるかが明確
- **4行目**: リリース予定 → 「今フォローしておく理由」を作る
- **5行目**: CTA。「フォローしといて」= 山田の文体。ターゲットを限定（店選びで消耗してる人）

### リンク欄
```
LINE公式のURL
```

### 固定ポスト（LINE登録導線）
```
グルメスワイプ、3月にベータ版出します

「店選びに30分かけるのもう嫌だ」って人向けに作ってる
動画スワイプで直感的に店が見つかるアプリ

先行で使いたい人はLINE登録しといてくれ
ベータ版の招待、LINEから先に送る👇

（リプライにLINEリンクを貼る）
```

**設計の解説:**
- LINE登録の動機 = 「先行招待がもらえる」（限定性）
- リンクはリプライに分離（アルゴリズム対策）
- 固定ポストなのでプロフィール訪問者全員が見る

## A/B/C 比較

| 項目 | A（推奨） | B | C |
|------|-----------|---|---|
| ターゲット明確度 | ◎ 「店選びで消耗してる人」 | ○ 「30分迷う人」 | ○ 命令形で明確 |
| 山田らしさ | ◎ カジュアル＆自然 | ○ やや真面目 | ◎ 強め・テンポ良い |
| フォロー動機 | ◎ 「フォローしといて」 | ○ 「気になる人はフォロー」 | ◎ 「損はさせない」 |
| 信頼感 | ○ エンジニアの記述あり | ◎ 8年目を強調 | ○ 括弧で補足 |
| 記憶に残る度 | ◎ 数字対比 | ○ | ◎ 命令形インパクト |

## 補足: やってはいけないこと

1. ❌ Bio内にLINEリンクのURLを直書き → フォロー率が下がる（目的が散る）
2. ❌ 「フォロー＆LINE登録お願いします！」→ 押し売り感
3. ❌ 名前を頻繁に変える → アルゴリズムのペナルティ報告あり
4. ❌ Bio内にハッシュタグ → 2026年は非推奨
5. ❌ 「〜をあなたに」「〜をお届け」→ 企業アカウント感
