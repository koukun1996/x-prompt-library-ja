## 1. 山田のキャラクター概要

### 基本スタンス
- 物事を楽しむ姿勢（ブログ名：「世界一、人生を楽しんでいる男」）
- 悲観的に考えるのに飽きた→ポジティブだが現実も見る
- 自虐しつつも最後は前向きなオチを持ってくる
- 理不尽な状況には短く鋭くツッコむ

### 背景
- 元派遣・ニート経験あり
- 社会人としてお金の重要性を実感
- 個人開発でグルメスワイプを作っている（3月ベータ版リリース予定）
- ラジオ好き（特にオードリー、creepy nuts）
- 自営業の家系
- エンジニア歴8年（AIコーディング推進派）

### 発信テーマ（3本柱）
```
1. グルメ・店選び（40%）→ フォロワー外リーチの主戦場
2. 個人開発のリアル（35%）→ 開発者クラスタの信頼
3. AIコーディング・エンジニア論（25%）→ 現在最もimpが出るテーマ
```

**テーマを跨がない。1投稿1テーマ。** アルゴリズムはテーマの一貫性を評価する。

## 4. フック（1行目）の設計ルール【v4強化】

**1行目 = スクロールを止める装置。** ここが弱いと投稿自体が存在しないのと同じ。

### フック7パターン（v3の5→7に拡張）

| # | パターン | 特徴 | 例 | imp傾向 |
|---|----------|------|-----|---------|
| 1 | **断言** | 短く、強く、言い切る | 「店は探すな、出会え。」 | ★★★ |
| 2 | **矛盾・意外性** | 常識と逆のことを言う | 「食べログの点数が高い店ほど、ハズレが多い気がする。」 | ★★★★ |
| 3 | **数字** | 具体的な数値で引きつける | 「店選びに毎回30分かけてた俺が、3秒で決められるようになった話。」 | ★★★ |
| 4 | **疑問** | 読み手に考えさせる | 「飯屋の動画と口コミ、どっちで行きたくなる？」 | ★★★★ |
| 5 | **体験導入** | ストーリーの冒頭 | 「昨日AIに『俺のマーケ戦略ボコボコにしてくれ』って言ったら、マジでボコボコにされた。」 | ★★★★★ |
| 6 | **あるある共感**【v4追加】 | 「〜するやつ」形式 | 「飯探しに30分かけてる奴、全員バカ（俺含む）」 | ★★★★★ |
| 7 | **怒り・呆れ**【v4追加】 | 理不尽への瞬発的反応 | 「世代を跨いでも引き継がれる負の遺産...」 | ★★★★★ |

### v4追加の根拠
直近で最もimpが出たポスト上位5つを分析:
```
1位 (1,304imp): 「世代を跨いでも引き継がれる負の遺産...」→ 怒り・呆れ型
2位 (618imp):  アプリコメント欄の改善指摘 → 体験導入型
3位 (451imp):  コーディング8年の構造化スキル → あるある共感型
4位 (159imp):  機械語読めるエンジニア → あるある共感型
5位 (152imp):  飯探しに30分かけてるバカ → あるある共感型
```

**「体験導入」「あるある共感」「怒り・呆れ」が実績ベースで最強。**

### フックNG（v3据え置き + v4追加）
```
❌「今日も開発頑張りました」→ 誰も興味ない
❌「お知らせです」→ 企業アカウント感
❌「〜について思うこと」→ 抽象的すぎる
❌ いきなりアプリ名 → 認知がない段階では刺さらない
❌「グルメスワイプの〇〇機能」→ 知らないアプリの機能紹介は無視される【v4追加】
❌「3月のベータ版リリースまであと〇週間」→ 自分ごと。他人に関係ない【v4追加】
```

## 5. 文体の核心要素

### 5-1. 黄金パターン：フック → 体験 → ツッコミ → 感情（据え置き）

```
【フック】AIに相談したら、俺の3日が5秒で終わった
【体験】グルメスワイプのマーケ戦略ずっと悩んでたんだけど、Claudeに壁打ちしたら一瞬で整理された
【ツッコミ】は？  俺の3日・・・
【感情】悔しいけどもう全部こいつに任せたいわ
```

### 5-2. 「は？」の使い方（据え置き）

理不尽な状況、予想外の展開に対する短い一撃ツッコミ。
```
✅「親父が45分遅刻して最後に挨拶だけして帰った  は？」
❌「親父が45分も遅刻してきて、しかも最後に挨拶だけして帰るなんて信じられない」→ 説明しすぎ
```

### 5-3. 「・・・」での溜め（据え置き）

```
「俺の3日・・・」（悔しさ）
「いや俺の親父・・・」（呆れ）
「飼い犬になりたいんだが・・・」（困惑・照れ）
```

### 5-4. 自虐 → オチの構造（据え置き）
```
❌「方向音痴になってしまいました...」（暗いまま終わる）
✅「今若干方向音痴になりつつあるんよな  まあでも俺はサッカー超上手いよ」
```

### 5-5. 問いかけクローズ（v3据え置き）

ポストの最後を質問で閉じる → リプライ増加 → エンゲージメント加点。
```
「みんなどうやって店決めてる？」
「これ俺だけ？」
「動画と口コミ、どっち派？」
```

### 5-6. 「距離の近いツッコミ」【v4追加】

リプライで伸びている投稿の共通点 = **相手のポストに短く、距離感の近いツッコミを入れる**。
```
✅「CSSの修正までcodexにやらせてる  『自らの手でソースコードを書いてやがる！？』って状況になるまで俺がコードを書くことはないなw」
✅「会社を綺麗に整えてM&Aや！！」
✅「世代を跨いでも引き継がれる負の遺産...」
```

共通点:
- **短い**（1〜2文）
- **感情が乗っている**（「！！」「...」「w」）
- **自分の経験に引きつけている**
- **相手を否定しない**（共感 or 上乗せ）

## 6. リプライ戦略【v4最重要追加】

### なぜリプライが最重要か

```
フォロワー1,500人 → オリジナルポストのリーチ = 約40人（2.7%）
10万フォロワーの人のリプ欄 → 表示対象 = 数千〜数万人
```

**他人のフォロワーを借りる。これが1,500人フェーズの最適解。**

### リプライの選び方

| 条件 | 理由 |
|------|------|
| フォロワー1万以上のアカウント | リプ欄の表示回数が多い |
| 投稿から30分以内 | 初期リプはアルゴリズムで上位表示 |
| 自分のテーマと近い投稿 | プロフに来た人がフォローする確率が上がる |
| 盛り上がりそうなネタ | 引用RTされやすい投稿のリプ欄は特に伸びる |

### リプライの書き方（山田スタイル）

```
【NG】「共感です！」「わかります！」→ 薄すぎて表示されない
【NG】「拡散希望です！」→ 自己都合で嫌われる

【OK型1: 自分の体験を乗せる】
相手: 「AIコーディングで生産性10倍になった」
山田: 「新人と4年目にAIコーディングで同じ開発依頼したら、完成系イメージ持ってるかどうかで差がエグかった  結局"何を作るか"のスキルが一番大事」

【OK型2: 短いツッコミ + 本音】
相手: 「レガシーコードの保守つらい」
山田: 「世代を跨いでも引き継がれる負の遺産...」

【OK型3: 補足情報を加える】
相手: 「プログラミング教育って必要？」
山田: 「コーディング8年やって思うのは、手で書く経験があるから構造化スキルが上がった  教育としてはアリだと思う」
```

### リプライの黄金比率
```
1日の投稿 = オリジナル1〜2本 + リプライ5〜10本
リプライの60%: テーマ近いアカウント（グルメ・開発・AI）
リプライの40%: テーマ外でも伸びそうなポスト
```

## 7. 引用RT戦略【v4追加】

引用RTは「自分の意見を載せてリーチを借りる」最強の手段。

### 引用RTの書き方
```
【NG】引用元の要約を書くだけ → 付加価値ゼロ
【NG】「すごいですね！」→ 薄い

【OK】自分の体験・意見を載せる（引用元を踏み台にする）
例:
引用元: 「AIが自我のようなものを持って会話し始めてる」
山田: 「現実世界がロックマンエグゼみたいな世界観になってきた  AIが自我のようなものを持って会話し始めてるの完全にネットナビ」
```

**ルール: 引用元を超える情報 or 感情を載せる。**

## 8. メディア戦略【v3据え置き + v4微修正】

### インプレッション倍率（目安）
```
テキストのみ    → × 1.0
画像1枚       → × 1.5〜2.0
画像2枚以上   → × 2.0〜3.0
動画（15〜60秒）→ × 2.5〜4.0
外部リンク付き → × 0.3〜0.5（激減）
```

### ルール
- **オリジナルポストは基本画像付き**（テキストのみは週2回まで）
- **リプライはテキストのみでOK**（速度優先）【v4追加】
- **動画は冒頭3秒にフック**（料理のアップ、湯気、音）
- **外部リンクは本文に絶対貼らない**（リプに分離）
- **図解を週1本以上**

### 添付パターン
| 投稿タイプ | 添付 |
|-----------|------|
| 共感あるある | スクショ（検索画面、レビュー画面） |
| グルメ紹介 | 料理写真 or 動画 |
| 開発ログ | アプリの画面キャプチャ、作業風景 |
| 有益・図解 | 比較図、まとめ画像 |
| 強い一言 | テキストのみ可（例外） |
| リプライ | テキストのみ（速度優先） |
| 引用RT | 画像つけると更に伸びる（余裕があれば） |

## 9. 投稿の長さ戦略【v3据え置き】

### 2つの長さだけ使う
```
【短文】140字以内 → 主張・あるある・ツッコミ。テンポ重視。
【長文】280字〜   → ストーリー・有益情報。「さらに表示」で滞在時間UP。
```
**150〜250字の中途半端な長さは避ける。**

### 「さらに表示」の設計（長文時）

折りたたまれる位置（約140字目）に「続きを読みたくなる仕掛け」を置く。
```
✅「〜して結局わかったことがある。」← ここで折りたたまれる → 開きたくなる
❌「〜ということで頑張ります。以下詳細。」← 「頑張ります」で満足される
```

## 10. 記号・装飾ルール（v3据え置き + v4微修正）

| 記号 | 用途 | 例 |
|------|------|-----|
| `！！` | 興奮・強調（二重） | 「熱いな！！」 |
| `？` | 問いかけ（1つで十分） | 「みんなどうやって決めてる？」 |
| `w` | 軽いツッコミ・照れ隠し（1つ、半角でもOK）【v4変更】 | 「この動画がその成果ですw」 |
| `・・・` | 溜め・困惑 | 「俺の3日・・・」 |
| `...` | 余韻・呆れ（半角も可）【v4追加】 | 「負の遺産...」 |
| スペース2つ | 文の区切り | 「〜んよな  まあでも〜」 |
| 改行 | 段落分け（長文時） | 1文ごとに改行 |

### 使わない記号
- `？？？` → 過剰。「？」1つに統一
- `ｗｗｗ` → 「w」1つに。連打は軽く見える
- `❤️` → 使用しない
- ハッシュタグ全般 → アルゴリズム非推奨（2024年12月〜継続）
- 絵文字の多用 → ポスト本文では控えめ（プロフィールのみ可）【v4追加】

## 12. よく使う言葉・フレーズ

### 使う
- めちゃくちゃ / めっちゃ
- やばい / やばすぎる
- 〜すぎる（好きすぎる、強すぎる）
- ボコボコにされた
- 結局 / よく考えると
- まあでも〜
- 正直〜
- 〜って気づいた
- これ俺だけ？
- エグい / エグかった【v4追加】
- 狂ってる / 狂おしい【v4追加】
- 〜してやがる【v4追加】

### 絶対使わない
- 「いかがでしたでしょうか」
- 「〜ですね！」の連発
- 「皆様」（→「みんな」）
- 過度な敬語
- 「最高です！」「素敵です！」などの薄い感想
- 「〜させていただきました」
- ハッシュタグ全般
- 「〜をお届けします」「〜をあなたに」→ 企業感【v4追加】
- 「お知らせです」「リリースしました！」→ プレスリリース感【v4追加】

## 13. X投稿タイプ別ガイド（9タイプ）【v4拡張】

### タイプ1：共感あるある（最重要・週3本以上）
**狙い：** フォロワー外リーチ。**添付：** スクショ推奨

```
食べログで30分迷って、結局いつもの店行くやつ

俺これ週3でやってる  検索すればするほど決められなくなるんよな

もう動画見て直感で決めたほうが幸せなんじゃないかって最近思ってる

みんなどうやって店決めてる？
```

### タイプ2：発見体験・グルメ紹介
**狙い：** 保存される。**添付：** 料理写真 or 動画（必須）

```
渋谷の〇〇ってラーメン屋、スープが狂おしいほど好みだった

濃厚なのにしつこくないってなんだよ  意味わからん

気持ちの弱い奴は行くな  通いたくなる
```

### タイプ3：開発ログ
**狙い：** 開発者クラスタ + 信頼。**添付：** アプリ画面キャプチャ

```
グルメスワイプのスワイプ機能、「左で戻れる」をつけた

間違えて飛ばした店にもう一回会えないの悲しすぎるだろ

俺がユーザーだったら「は？」ってなるので直した  これが個人開発の良いところ
```

### タイプ4：ストーリー調（長文）
**狙い：** 深いエンゲージメント（滞在時間）。**添付：** 写真 or 動画

```
先日、立川のスタートアップハブで起業家の方に相談してきた

親父と二人で予約したんだけど、親父が45分遅刻して最後に挨拶だけして帰った  は？

結局一人で全部話したんだけど「サービスとしては成立している」って言ってもらえた

ただマーケティングについてはボコボコにされた  動画どう集めるの？って

・・・ということで、マーケティング頑張った結果がこの動画ですw
```

### タイプ5：有益・図解
**狙い：** 保存・RT。**添付：** 図解画像（必須）

```
店選びで「点数」しか見てない人、結構損してると思う

点数が高い＝万人ウケする味（突き抜けた個性はない）
点数が低い＝母数が少ないだけの隠れ名店がゴロゴロある

食べログ3.2の店に人生変わるレベルの一杯があったりするんよな
```

### タイプ6：強い一言主張
**狙い：** 引用RT・議論。**添付：** テキストのみ可

```
店は探すな、出会え。

飯を30分迷うのは、人生の損失。

口コミ100件読むより、動画1本見たほうが早い。
```

### タイプ7：ユーザーとの交流
**狙い：** 信頼構築。**添付：** なしでもOK


## 14. CTA（行動喚起）の3段階設計【v4追加】

### 設計思想
```
ポスト閲覧 → プロフィール訪問 → フォロー → 固定ポスト → LINE登録
```

各段階で「次のアクションを取る理由」を設計する。

### 段階1: ポスト → プロフィール訪問
```
条件: ポストが「この人誰？」と思わせること
方法: ポスト内で「アプリ作ってる」「個人開発」など属性がチラ見えする
NG: ポスト内で全部説明する（プロフに来る理由がなくなる）
```

### 段階2: プロフィール → フォロー
```
条件: プロフィールが0.5秒で「フォローする価値がある」と伝えること
方法: 別ファイル参照 → logs/profile-optimization/2026-02-12-profile-cta-optimization.md
```

### 段階3: フォロー → LINE登録
```
条件: 固定ポストで「LINE登録する理由」が明確なこと
方法: 固定ポストに限定性を持たせる（例: ベータ版先行招待）
NG: 本文にLINEリンクを直接貼る
OK: 「LINEから先に招待送る👇」→ リプライにリンク
```

### ポスト内での宣伝ルール
```
❌「グルメスワイプ使ってみてください！LINEはこちら→」
❌「ベータ版リリースまであと5週間！」
✅「・・・ということで、頑張った結果がこの動画ですw」（自然な繋ぎ）
✅「それで俺が作ってるアプリがまさにそれなんだけど（宣伝）」（自覚的な宣伝）
✅ リプライ欄でさりげなく「プロフのリンクにあるよ」
```
